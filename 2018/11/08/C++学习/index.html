C++学习

 · 2018-11-8 · 次阅读

  1. Std::out << “hello world” << std::endl
  2. 环境只是决定了代码的编译方式
  3. 当编译单个文件时，出现obj，当编译整个项目时，linker将obj文件组合起来变成exe

#编译
1.将所有的C++文件编译为单个obj文件
2.在编译之前会先执行#中的语句，我们可以在#中使用条件结构以及定义数据

 #if 1 

 #end if

#链接

链接就是将许多单个的obj文件变为一个可执行exe文件

#变量
变量类型：bool int float double char wchar_t（宽字符）
变量修饰符：long short signed unsigned

  • 像size_t这种数据类型，其本质就是typedef int，我们只是为了定义一种变量，能用名字来表示他的功能，为什么要这么做呢？这样更方便阅读源码
  • void的字面意思是“无类型”，void 则为“无类型指针”，void 可以指向任何类型的数据，改半天是个万金油

#函数
auto b = ->int{return a*b;};

##函数指针
定义：函数指针就是指向函数入口（地址）的指针

int addTwo(int a,int b){
    return a+b;
}
int (*add) (int,int) = addTwo;//赋值一个函数指针
int result = (*add)(2,3);//调用一个函数指针

##loop
while for do….while

  • break：结束循环
  • continue：结束当前循环，进入下一条循环
  • goto

##control flow
if if….else switch
Exp1 ? Exp2 : Exp3;

#运算符

  • 流插入运算符 <<
  • 流提取运算符 >>

1.当我们输出的变量类型改变时，需要用流插入运算符重载 

cout << "我今年" << 18 << endl

2.当我们输出自己定义的类型时候，必须对他们进行重载

istream & operator >> (istream &, 自定义类 &);
ostream & operator << (ostream &, 自定义类 &);
  • 作用域符号 ：：
    （1）作用域符号”::“的前面一般是类名称，后面一般是该类的成员名称，C++为避免不同的类有名称相同的成员而采用作用域的方式进行区分。
    （2）全局变量符号 ::char
    （3）作用域分解运算符 “比如声明了一个类A，类A里声明了一个成员函数void f()，但没有在类的声明里给出f的定义，那么在类外定义f时，就要写成void A::f()，表示这个f()函数是类A的成员函数。例如：
Thread::Thread(char* threadName)

#指针

  • 字符串常量的本质表现是代表它的第一个字符的地址
    char *Buffer = “China” 其实是把字符串第一个字符的地址赋予给Buffer

type_name 这种type的指针
&取地址运算符 ()取值运算符

    int a = 5
    int* ptr = &a;
    std::cout << ptr << std::endl;
    std::cout << *(ptr) << std::endl;

#数组
type arrayName [ arraySize ];

    int a[] = {1,2,3,4};
    std::cout << a << std::endl;
    std::cout << &a[0] <<std::endl;
    std::cout << a[0] << std::endl;
    std::cout << *(a) <<std::endl;

#存储类
extern 声明（引用）其他的cpp文件的函数或全局变量（不需要include）,翻译为来自外部的

int a = 5;
extern void print_();
int main(int argc, const char * argv[]) {
    // insert code here...
    print_();
}

#include <stdio.h>
#include "iostream"
extern int a;
void print_(){
    std::cout << a << std::endl;
}

线程

fork() //创建一个和自己一模一样的子进程然后运行，共享数据块和代码段，**子进程返回0，
父进程返回子进程的进程号**。
vfork() //创建一个子进程然后运行。vfork 保证子进程先运行，在调用exec 或exit 之前与父进程**数据块**是共享的,在它调用exec 或exit 之后父进程才可能被调度运行。 父进程可能出现死锁
wait() //等待某个线程执行完成

宏定义

  • #在预处理阶段被安排
#define            定义一个预处理宏
#undef            取消宏的定义


//条件编译
#if                   编译预处理中的条件命令，相当于C语法中的if语句
#ifdef              判断某个宏是否被定义，若已定义，指针if结构中的语句
#ifndef            与#ifdef相反，判断某个宏是否未被定义
#elif                若#if, #ifdef, #ifndef或前面的#elif条件不满足，则执行#elif之后的语句，相当于C语法中的else-if
#else              与#if, #ifdef, #ifndef对应, 若这些条件不满足，则执行#else之后的语句，相当于C语法中的else
#endif             #if, #ifdef, #ifndef这些条件命令的结束标志.
defined         　与#if, #elif配合使用，判断某个宏是否被定义

#define ngx_string(str)     { sizeof(str) - 1, (u_char *) str }
//宏定义的本质还是将前面的替换为后面的
