软件测试

软件测试就是尽一切手段让它发生错误

  • 测试用例：给出一个输入，检测系统的输出
  • 前置条件：测试之前需要满足的条件

测试分类：

  • 静态(static)
  • 功能(functional)
  • 结构(Structure)

白盒测试

  • 覆盖率(statement coverage)：测试用例覆盖的代码块
  • 分支覆盖率(branch coverage)：测试用例覆盖的判断语句
  • 条件覆盖率(condition coverage)：测试用例覆盖的判断语句(一个判断语句中的条件需要被拆分)

黑盒测试

  • 等价类划分：枚举是strong，单独抽一个是weak；域是每一个输入可能的取值域；需要划分域，再将域划分
  • 边界值：从等价类中的边缘取值

基于模型的测试

建模就是给出一个事物的抽象，软件工程中所学的图表只是一些常用的抽象方式。建模能帮助生成、简化代码，验证系统的正确

基于模型的测试可以在代码未完成构建之前进行测试设计，是一种轻量级的测试；系统化的从模型生成一组测试用例，使用这组测试用例测试待测软件系统；即从模型中设计、生成测试用例

行为模型

step：

  • Operations
  • Classes
  • Objects（init data）
  • State Machine（state diagram）
  • Object constraint language（OCL）

基于模型的健壮性测试(MBRT)：测试系统在异常输入或者有压力的环境是否还能保持正确

测试用例的选择

  • 考虑在判断时，需要的距离，例如if(x>y). （x=1，y=2）距离是1，（x=1，y=101）距离是100

商务智能

什么是商务智能？

商业智能（Business Intelligence，简称：BI），又称商业智慧或商务智能，指用现代数据仓库技术、线上分析处理技术、数据挖掘和数据展现技术进行数据分析以实现商业价值。

　　商业智能作为一个工具，是用来处理企业中现有数据，并将其转换成知识、分析和结论，辅助业务或者决策者做出正确且明智的决定。是帮助企业更好地利用数据提高决策质量的技术，包含了从数据仓库到分析型系统等。

数据和知识之间的区别？

我们从大量的数据中学习，在学习中掌握新的知识。

什么是数据库和数据管理系统，他们之间的区别？

数据库就是一组数据的集合，数据管理系统帮助我们管理这组数据集合
